The Problem     (January 2016)

Once more, Jo asked me to help him on a new artist idea. What he likes now is to create a group of steel statues where each one has it's own brightness regulated light of 220V. Sounds like aother mcu application.

Yes, but there is one little problem: the mcu that I konw so far are not specially equiped for use with triacs. To implement the proper triac ignition timing within a half-sine of the 220V current,  I used one timer per connected triac to keep the implementation easy. At least on an old, slow 8 bit mcu this is probable the best design, while with a fast 32 bit mcu and a real time operating system the implementation could be held in a appropriate clean design even with one single timer for all triacs together. But the mcu application I am developing, even when later I will use a 32 bit mcu, should also be able to run on a small 8 bit mcu with only a few timers available (typically 2 or 3).

Now instead of looking for an existing IC that would do the triac driver job, for studying reasons, I plan to develop a multi triac driver on an FPGA. Since I already own an Altera cyclone II development kit, I plan to develop a VHDL design on this cyclone II using Altera's quartus II IDE version 13.0.1sp1 Web edition (64 bit).
